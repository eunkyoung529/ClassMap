## **1. 커밋 컨벤션**

커밋 메시지는 다음 규칙을 따릅니다:

```
[타입] 모듈명: 메시지 내용
```

- **타입** (소문자):
    - feat: 새로운 기능 추가
    - fix: 버그 수정
    - docs: 문서 수정
    - style: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
    - refactor: 리팩토링 (기능 추가/변경 없음)
    - test: 테스트 코드 추가/수정
    - chore: 빌드 작업, 의존성 관리 등 기타 작업

예시:

```
feat member: 회원 가입 기능 구현
fix auth: 로그인 시 인증 오류 수정
```
